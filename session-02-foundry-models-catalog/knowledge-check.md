# Knowledge Check: Foundry Models I — Catalog & Comparison

Answer these before checking the key. Works as a self-test, a LinkedIn poll, or a quiz carousel.

1. What are the filter categories in the Foundry model catalog's filter bar?
2. True or false: every model in the catalog supports every deployment type (standard, provisioned throughput, serverless).
3. What's the difference between a model's benchmark score and its licensing terms, in terms of what each tells you?
4. If a model card shows a region restriction, what does that affect?
5. Name one reason a lower-benchmark model might still be the right choice for a task.
6. What are the four criteria used in the catalog's Model leaderboards?

<details>
<summary>Answer key</summary>

1. Collections, Industry, Capabilities, Deployment options, Inference tasks, Fine-tuning tasks, and Licenses.
2. False — deployment type availability varies by model and by region.
3. Benchmark scores estimate task performance on standardized tests; licensing terms govern what you're legally allowed to do with the model's outputs and whether it can be fine-tuned or used commercially.
4. Which Azure regions you can deploy that model into — this can conflict with your project's existing region if you're not paying attention.
5. Cost, latency, or licensing terms that fit the task better — a smaller model that's "good enough" and cheaper often beats a larger one for high-volume, low-complexity tasks.
6. Quality, Safety, Cost, and Throughput.

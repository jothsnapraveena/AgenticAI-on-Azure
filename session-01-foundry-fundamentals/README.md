# Session 1: Foundry Fundamentals

![Repository architecture](images/1.png)

## What this session covers

The Microsoft Foundry portal, the hub/project structure underneath it, and where role-based access control fits before you touch a single model or agent. This is the orientation session — everything from Session 2 onward assumes you're comfortable navigating Foundry and know what a "project" actually is.

## Why it matters

Foundry consolidates what used to be separate Azure AI services (Azure OpenAI, Azure AI Search, Azure AI Studio) into one resource provider namespace. If you've used any of those before, some concepts will look familiar but sit in a different place now. Starting here avoids the confusion of hitting "hub," "project," or "connection" in a later session's docs without knowing what they mean structurally.

## What you'll gain

By the end of this session, you will be able to:
- Explain the relationship between a Foundry hub and a Foundry project
- Create a project in the Foundry portal (ai.azure.com)
- Identify where RBAC is applied at the hub vs. project level
- Locate the Operate tab and explain what it's for (previewed here, used in depth in Session 7 and 8)

## Prerequisites

- Active Azure subscription
- Entra ID account with Contributor or Owner access on the subscription (or a resource group scoped to you)
